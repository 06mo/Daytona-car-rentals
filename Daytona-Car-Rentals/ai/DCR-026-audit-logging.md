# DCR-026 — Audit Logging System Design

**Owner:** Claude (design)
**Implementer:** Codex (DCR-027)
**Status:** Complete

---

## Objective

Provide a durable, queryable trail of every significant state change in the system: booking lifecycle transitions, payment events, document reviews, and admin actions. The log must be append-only from the application's perspective, readable only by admins, and must never block the primary operation that triggered it.

---

## Collection Schema

### Top-level collection: `audit_logs`

Each document is an immutable event record written once and never updated.

```
audit_logs/{logId}
```

| Field | Type | Description |
|---|---|---|
| `id` | `string` | Auto-generated Firestore document ID |
| `entityType` | `AuditEntityType` | The type of resource affected |
| `entityId` | `string` | The ID of the specific resource |
| `action` | `AuditAction` | What happened |
| `actorId` | `string` | `userId` of the person who triggered it, or `"system"` for webhook/automated events |
| `actorRole` | `AuditActorRole` | Role of the actor at the time of the event |
| `timestamp` | `Timestamp` | When the event occurred (server-set) |
| `before` | `Record<string, unknown>` (optional) | Snapshot of relevant fields before the change |
| `after` | `Record<string, unknown>` (optional) | Snapshot of relevant fields after the change |
| `metadata` | `Record<string, unknown>` (optional) | Additional context — see per-action examples below |

## Service: `lib/services/auditService.ts`

```typescript
import "server-only";

export async function logAuditEvent(input: CreateAuditLogInput): Promise<void>
```

### Rules for `logAuditEvent`

1. **Best-effort only.** Wrap the Firestore write in try/catch. Log errors to `console.error` but never throw. Audit failures must not abort the primary operation.
2. **Use `addDocument`** from `lib/firebase/firestore.ts` to write to `audit_logs`. The `id` is auto-generated by Firestore.
3. **Server-set timestamp.** Do not accept `timestamp` from the caller. Set it inside `logAuditEvent` as `new Date()`.
4. **`FirebaseConfigError` is silently swallowed.** In the dev/mock path, audit logging is a no-op.

```typescript
export async function logAuditEvent(input: CreateAuditLogInput): Promise<void> {
  try {
    await addDocument("audit_logs", {
      ...input,
      timestamp: new Date(),
    });
  } catch (error) {
    console.error("[audit] Failed to write audit event:", input.action, error);
  }
}
```

## Integration Points

Call `logAuditEvent` after every successful write. Do not await it inline if it would add latency to the critical path — fire-and-forget using `void logAuditEvent(...)` is acceptable where the primary return value is not the audit record.

### `lib/services/bookingService.ts`

| Function | Action | before | after | metadata |
|---|---|---|---|---|
| `createBooking` (after transaction) | `booking.created` | — | `{ status, paymentStatus, vehicleId, startDate, endDate }` | `{ vehicleId, startDate, endDate }` |
| `updateBookingStatus` | `booking.status_changed` | `{ status: prevStatus }` | `{ status: newStatus }` | `{ adminNotes }` if provided |
| `cancelBooking` | `booking.cancelled` | `{ status: prevStatus, paymentStatus: prevPaymentStatus }` | `{ status: "cancelled" }` | `{ cancelledBy, cancellationReason }` if provided |

For `createBooking` and `cancelBooking`, the `actorId` and `actorRole` are not available inside the service function. Accept them as optional parameters: `actorId?: string`, `actorRole?: AuditActorRole`. Default `actorId` to `"system"` and `actorRole` to `"system"` if not provided.

### `lib/services/documentService.ts`

| Function | Action | metadata |
|---|---|---|
| `upsertUserDocument` | `document.submitted` | `{ documentType, fileName, mimeType }` |
| `reviewUserDocument` (approve) | `document.approved` | `{ documentType, reviewedBy }` |
| `reviewUserDocument` (reject) | `document.rejected` | `{ documentType, reviewedBy, rejectionReason }` |

`entityId` = `userId` (the customer whose document was acted on), not the document ID.
`actorId` = `reviewedBy` (admin's userId) for review events; `userId` (customer) for submit events.

### `lib/services/userService.ts`

| Function | Action | before | after |
|---|---|---|---|
| `updateUserVerificationStatus` | `user.verification_status_changed` | `{ verificationStatus: prevStatus }` | `{ verificationStatus: newStatus }` |

`actorId` defaults to `"system"` — this is always triggered as a side-effect of a document review, not directly by a user action.

### `lib/stripe/webhooks.ts`

| Event | Action | metadata |
|---|---|---|
| `payment_intent.succeeded` | `payment.succeeded` | `{ stripePaymentIntentId, amount, currency }` |
| `payment_intent.payment_failed` | `payment.failed` | `{ stripePaymentIntentId, failureMessage }` |
| `charge.refunded` | `payment.refunded` | `{ stripePaymentIntentId, amountRefunded }` |

`entityType` = `"payment"`, `entityId` = `bookingId` (looked up via `findBookingByPaymentIntentId`).
`actorId` = `"system"`, `actorRole` = `"system"`.

### `lib/stripe/server.ts`

| Function | Action | metadata |
|---|---|---|
| `createPaymentIntentForBooking` | `payment.intent_created` | `{ stripePaymentIntentId, depositAmount, vehicleId }` |

`entityType` = `"payment"`, `entityId` = `bookingId`.

## Firestore Security Rules

Add to `firestore.rules`. Audit logs are server-written only — no client reads or writes.

```
match /audit_logs/{logId} {
  allow read: if request.auth != null && request.auth.token.role == "admin";
  allow write: if false;
}
```

## Indexes (`firestore.indexes.json`)

Add these three composite indexes:

```json
{
  "collectionGroup": "audit_logs",
  "queryScope": "COLLECTION",
  "fields": [
    { "fieldPath": "entityId", "order": "ASCENDING" },
    { "fieldPath": "timestamp", "order": "DESCENDING" }
  ]
},
{
  "collectionGroup": "audit_logs",
  "queryScope": "COLLECTION",
  "fields": [
    { "fieldPath": "actorId", "order": "ASCENDING" },
    { "fieldPath": "timestamp", "order": "DESCENDING" }
  ]
},
{
  "collectionGroup": "audit_logs",
  "queryScope": "COLLECTION",
  "fields": [
    { "fieldPath": "entityType", "order": "ASCENDING" },
    { "fieldPath": "action", "order": "ASCENDING" },
    { "fieldPath": "timestamp", "order": "DESCENDING" }
  ]
}
```

## Query Helpers (for DCR-027 to implement in `auditService.ts`)

```typescript
// All events for a specific entity (e.g., all events on booking "abc123")
export async function getAuditLogsForEntity(
  entityId: string,
  limit = 50,
): Promise<AuditLog[]>

// All actions taken by a specific admin
export async function getAuditLogsForActor(
  actorId: string,
  limit = 50,
): Promise<AuditLog[]>
```

Both use `listDocuments("audit_logs", { filters: [...], orderBy: [{ field: "timestamp", direction: "desc" }], limit })`.
